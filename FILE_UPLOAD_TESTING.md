# 🧪 دليل اختبار نظام رفع الملفات المتعددة

## ✅ الاختبارات المتاحة

### 1. اختبار الاتصال الأساسي
```powershell
# تحقق من أن الخوادم تعمل
docker-compose ps

# يجب أن ترى:
# - backend-1: HEALTHY ✅
# - frontend-1: HEALTHY ✅ (بعد ثوانٍ قليلة)
# - database-1: HEALTHY ✅
```

### 2. اختبار Backend Upload API
```powershell
# تحقق من أن endpoint الرفع يعمل
docker-compose logs backend | Select-String "running"

# يجب أن تراه:
# 🚀 Server running on http://0.0.0.0:5000
```

### 3. الدخول إلى التطبيق
```
انقر: http://localhost:3030
⬆️ يجب أن ترى صفحة التحميل
```

### 4. الذهاب إلى صفحة الورشة
```
http://localhost:3030/manage/workshop
⬆️ يجب أن ترى قائمة الأوامر
```

### 5. اختبار رفع الملفات
```
✅ خطوات الاختبار:
1. اختر أمر من مرحلة "التجهيز"
2. انقر على الأمر لفتح التفاصيل
3. ابحث عن قسم "📁 ملفات التصميم"
4. انقر على "اختر الملفات ➕"
5. اختر ملف من جهازك (أي نوع)
6. انقر "فتح" (Open)

❓ النتائج المتوقعة:
✅ يجب أن تراها في قائمة الملفات
✅ يجب أن تراها في Browser Console (Ctrl+Shift+K)
✅ يجب أن تراها في Docker Logs
```

### 6. اختبار حفظ البيانات
```
✅ خطوات الاختبار:
1. أضف مقاسات (مثال: 20x30 سم)
2. أضف نوع خامة (مثال: خشب)
3. أضف ملاحظات
4. انقر "💾 حفظ البيانات"

❓ النتائج المتوقعة:
✅ ظهور رسالة نجاح
✅ تحديث البيانات في Backend
✅ حفظ في MongoDB
```

### 7. اختبار حذف الملفات
```
✅ خطوات الاختبار:
1. انقر على 🗑️ بجانب أي ملف (فقط في مراحل التجهيز/التصميم)
2. يجب أن يختفي الملف

❌ المتوقع عدم ظهور الزر 🗑️:
- في مرحلة التشغيل
- في مرحلة التسليم
```

### 8. اختبار التحميل
```
✅ خطوات الاختبار:
1. انقر على ⬇️ بجانب أي ملف
2. يجب أن يتم تحميل الملف

⚠️ ملحوظة: قد يكون بطيئاً إذا كان الملف كبيراً
```

## ⚙️ اختبار متقدم

### عرض الملفات المرفوعة في النظام
```powershell
docker-compose exec backend find /app/uploads -type f | wc -l
# يعرض عدد الملفات
```

### اختبار حجم الملفات
```powershell
docker-compose exec backend du -sh /app/uploads/
# يعرض الحجم الإجمالي
```

### فحص رسائل الخطأ
```powershell
docker-compose logs backend | Select-String "Error|error|❌"
```

## 🎯 نقاط الاختبار الحرجة

| النقطة | الفحص | النتيجة المتوقعة |
|-------|------|-----------------|
| رفع ملف واحد | اختر ملف واحد | ✅ يظهر في القائمة |
| رفع ملفات متعددة | اختر 3+ ملفات | ✅ تظهر جميعاً |
| حذف الملف | انقر 🗑️ | ✅ يختفي الملف |
| التحميل | انقر ⬇️ | ✅ يتم التحميل |
| حفظ البيانات | انقر 💾 | ✅ رسالة نجاح |
| حجم الملف | رفع ملف > 50MB | ❌ فشل الرفع |
| نوع الملف | أي نوع | ✅ يعمل |
| صفحة أخرى | افتح الأمر نوع أخرى | ✅ البيانات محفوظة |

## 🚨 حل المشاكل الشائعة

### المشكلة: الملف لا يرفع
```
✅ الحل:
1. تحقق من الحجم < 50 MB ✓
2. تحقق من نوع الملف (أي نوع يعمل) ✓
3. افتح Console (F12) وشاهد الخطأ ✓
4. تحقق من backend logs:
   docker-compose logs backend
```

### المشكلة: الملف يختفي بعد التحديث
```
✅ الحل:
1. قد لم تكن قد حفظت الأمر
2. انقر 💾 حفظ البيانات
3. انتظر رسالة النجاح
4. ثم حدث الصفحة
```

### المشكلة: الزر الأحمر ❌
```
✅ الحل:
1. افتح Console (F12)
2. شاهد رسالة الخطأ الدقيقة
3. إذا كان CORS:
   تأكد من NEXT_PUBLIC_API_URL
4. إذا كان Backend:
   docker-compose restart backend
```
