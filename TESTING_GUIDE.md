# 🧪 دليل اختبار تحديث البيانات

## تجهيز البيئة  

### 1️⃣ تشغيل الخوادم
```bash
# في Terminal 1 - MongoDB (اختياري إذا تثبيت محلياً)
mongod

# في Terminal 2 - Backend
cd backend
npm install
npm start  # أو npm run dev للتطوير

# في Terminal 3 - Frontend
cd frontend
npm install
npm run dev
```

### 2️⃣ الانتظار حتى التشغيل الكامل
- Backend: `http://localhost:5000` ✅
- Frontend: `http://localhost:3000` ✅
- MongoDB: `mongodb://localhost:27017` ✅

---

## 🧪 سيناريوهات الاختبار

### السيناريو 1: إضافة منتج جديد ✅

**الخطوات:**
1. اذهب إلى `http://localhost:3000/manage/products`
2. امأ النموذج بالبيانات التالية:
   - **اسم المنتج:** "هدية خشب مخصصة"
   - **السعر:** 150
   - **القسم:** ليزر
   - **الصورة:** اختر صورة من جهازك (أو استخدم رابط)

3. اضغط زر "نشر المنتج 🚀"

**النتيجة المتوقعة:**
- ✅ ظهور رسالة نجاح
- ✅ ظهور المنتج الجديد في قائمة المنتجات **فوراً**
- ✅ عدم الحاجة لتحديث الصفحة
- ✅ يجب أن يكون المنتج مرئي عند دخول صفحة أخرى والعودة

**التحقق من النجاح:**
```javascript
// اذهب إلى أداة المطور (F12 → Console)
fetch('http://localhost:5000/api/products')
  .then(r => r.json())
  .then(d => console.log(d.data))
```

---

### السيناريو 2: حذف منتج ✅

**الخطوات:**
1. ابقَ في صفحة `/manage/products`
2. ابحث عن منتج تريد حذفه
3. اضغط زر "حذف المنتج 🗑️" بجانبه
4. أكد الحذف من النافذة المنبثقة

**النتيجة المتوقعة:**
- ✅ اختفاء المنتج من القائمة **فوراً**
- ✅ ظهور رسالة نجاح
- ✅ عدم ظهور المنتج عند التحديث (F5)

**التحقق من النجاح:**
```javascript
// تحقق من قاعدة البيانات
fetch('http://localhost:5000/api/products')
  .then(r => r.json())
  .then(d => {
    const count = d.data.length;
    console.log(`عدد المنتجات: ${count}`);
  })
```

---

### السيناريو 3: عرض المنتجات على الصفحة الرئيسية ✅

**الخطوات:**
1. اذهب إلى `http://localhost:3000`
2. لاحظ قائمة المنتجات

**النتيجة المتوقعة:**
- ✅ جميع المنتجات يجب أن تظهر بشكل صحيح
- ✅ يجب أن تظهر الصور بشكل صحيح
- ✅ يجب أن تكون الأسعار صحيحة
- ✅ يجب أن تعمل ميزة البحث والفلترة

**اختبار الفلترة:**
1. اختر قسم معين من القائمة (ليزر / طباعة / هدايا)
2. تحقق من أن المنتجات المعروضة تناسب القسم اختياره

---

### السيناريو 4: رفع الصور ✅

**الخطوات:**
1. اذهب إلى `/manage/products`
2. في قسم "الصورة"، اختر صورة من جهازك
3. تحقق من أن الصورة تظهر في المعاينة

**النتيجة المتوقعة:**
- ✅ ظهور معاينة الصورة
- ✅ حفظ رابط الصورة أثناء الإضافة
- ✅ ظهور الصورة في المنتج النهائي

**شروط صحة رفع الصورة:**
- ✅ نوع الملف: JPEG, PNG, GIF, WebP
- ✅ حجم الملف: أقل من 10MB
- ✅ البيانات الوصفية: صحيحة (لا أحرف خاصة)

---

### السيناريو 5: الإحصائيات والداشبورد ✅

**الخطوات:**
1. اذهب إلى `http://localhost:3000/manage` (لوحة التحكم)
2. لاحظ الأرقام في الإحصائيات

**النتيجة المتوقعة:**
- ✅ عدد الطلبات يجب أن يكون صحيح
- ✅ عدد الطلبات المعلقة يجب أن يكون صحيح
- ✅ عدد الطلبات تحت الإنتاج صحيح
- ✅ الأرقام يجب أن تتحدث تلقائياً كل 20 ثانية

**التحقق من النجاح:**
```javascript
// تحقق من عدد الطلبات
fetch('http://localhost:5000/api/orders')
  .then(r => r.json())
  .then(d => {
    console.log(`إجمالي الطلبات: ${d.total}`);
    console.log(`الطلبات الجديدة: ${d.data.filter(o => o.status === 'جديد').length}`);
  })
```

---

### السيناريو 6: إضافة طلب جديد ✅

**الخطوات:**
1. اذهب إلى الصفحة الرئيسية
2. اختر منتج و اضغط "إضافة للسلة" أو "شراء الآن"
3. أكمل نموذج الشراء بالبيانات التالية:
   - **الاسم:** اسمك الكامل
   - **الموبايل:** رقم هاتفك
   - **المحافظة:** اختر محافظة
   - **المدينة:** أدخل اسم المدينة
   - **العنوان:** أدخل العنوان

4. اضغط "إتمام الطلب"

**النتيجة المتوقعة:**
- ✅ ظهور رسالة نجاح مع كود الطلب
- ✅ يجب أن يظهر الطلب في `/manage/orders`
- ✅ يمكن تتبع الطلب من الصفحة الرئيسية

## 🔧 استكشاف الأخطاء

### خطأ: "فشل في جلب المنتجات"

**السبب المحتمل:**
- ❌ الخادم لا يعمل
- ❌ قاعدة البيانات لا متصلة
- ❌ مشكلة في الاتصال

**الحل:**
```bash
# تحقق من الخادم
curl http://localhost:5000/health

# إعادة تشغيل الخادم
npm start

# تحقق من MongoDB
mongo

# إعادة تشغيل كل شيء
npm install
npm start
```

---

### خطأ: "الصورة لم تُرفع"

**السبب المحتمل:**
- ❌ نوع الملف غير صحيح
- ❌ حجم الملف أكبر من 10MB
- ❌ حدث خطأ في الاتصال

**الحل:**
```bash
# اختبر رفع الصورة يدويّاً
curl -X POST \
  -F "file=@/path/to/image.jpg" \
  http://localhost:5000/api/upload

# تحقق من مجلد uploads
ls -la backend/uploads/

# تأكد من أن الأذونات صحيحة
chmod 755 backend/uploads/
```

---

### خطأ: "البيانات لم تتحدث بعد الإضافة"

**السبب المحتمل:**
- ❌ الكود لا يعيد جلب البيانات من الخادم
- ❌ مشكلة في الاتصال بـ API
- ❌ مشكلة في معالجة الاستجابة

**الحل:**
```javascript
// افتح أداة المطور (F12 → Console)
// تحقق من الأخطاء
console.error
```

## 📋 قائمة التحقق النهائية

قبل إطلاق الإنتاج، تأكد من:

- ✅ جميع المنتجات تعرض بشكل صحيح
- ✅ إضافة منتج يحديث القائمة فوراً
- ✅ حذف منتج يحديث القائمة فوراً
- ✅ رفع الصور يعمل بشكل صحيح
- ✅ الإحصائيات تُظهر البيانات الصحيحة
- ✅ الطلبات تُحفظ بشكل صحيح
- ✅ التتبع يعمل بشكل صحيح
- ✅ لا أخطاء في وحدة تحكم المتصفح (F12)
- ✅ الأداء مقبول (لا تأخير غير معقول)
- ✅ التصميم يعرض بشكل صحيح على جميع الأجهزة
